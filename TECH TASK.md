# Техническое задание
## Веб-приложение интернет-магазина электроники «TechnoStore»

---

## 1. Общие сведения

| Параметр | Значение |
|---|---|
| Название проекта | TechnoStore |
| Тип приложения | Веб-приложение (интернет-магазин) |
| Назначение | Продажа электроники через интернет |
| Разработчик | Евгений (дипломная работа) |
| Версия документа | 1.1 |

---

## 2. Цель и назначение

Разработать полнофункциональное веб-приложение интернет-магазина электроники, обеспечивающее:
- удобный просмотр и поиск товаров для покупателей;
- оформление заказов с оплатой наличными при получении;
- управление товарами, заказами и пользователями через панель администратора;
- полную адаптивность интерфейса для десктопных и мобильных устройств.

---

## 3. Технический стек

| Категория | Технология |
|---|---|
| Frontend | Next.js 15 (App Router), TypeScript, Tailwind CSS v4, Zustand |
| Backend | Next.js API Routes / Server Actions |
| ORM | Prisma |
| База данных | Vercel Postgres |
| Кэширование | Upstash Redis |
| Runtime | Bun |
| Деплой | Vercel |
| SMS-провайдер | Mock (dev) / Messaggio, BudgetSMS (prod) |

---

## 4. Роли пользователей

### 4.1 Покупатель (неавторизованный)
- Просмотр каталога и карточек товаров
- Поиск и фильтрация товаров
- Регистрация и вход в аккаунт через номер телефона

### 4.2 Покупатель (авторизованный)
- Всё из п. 4.1
- Добавление товаров в корзину
- Оформление заказов
- Просмотр истории заказов
- Управление профилем (имя, номер телефона, адрес доставки)
- Отслеживание статуса заказа

### 4.3 Администратор
- Управление товарами (создание, редактирование, удаление)
- Управление категориями и брендами
- Просмотр и управление заказами (смена статусов)
- Просмотр списка пользователей

---

## 5. Функциональные требования

### 5.1 Авторизация и регистрация

- Регистрация через номер телефона + SMS-код
- Вход через номер телефона + SMS-код (каждый вход = новый код)
- Хранение сессии (JWT в cookie)
- Защищённые роуты для авторизованных пользователей и администраторов
- SMS-коды в режиме разработки логируются в консоль (mock)

### 5.2 Каталог товаров

- Список товаров с пагинацией
- Карточка товара:
    - название
    - галерея фото (несколько изображений)
    - цена
    - краткое описание
    - характеристики (CPU, RAM, Storage и т.д. в зависимости от категории)
    - наличие на складе
    - кнопка «В корзину»
- Категории товаров (ноутбуки, смартфоны, планшеты, аксессуары и т.д.)
- Бренды

### 5.3 Поиск и фильтрация

- Поиск по названию товара
- Фильтрация по:
    - категории
    - бренду
    - диапазону цен
    - наличию на складе
- Сортировка: по цене (возр./убыв.), по новизне, по популярности

### 5.4 Корзина

- Доступна только для авторизованных пользователей
- Добавление товара в корзину
- Изменение количества товара
- Удаление товара из корзины
- Отображение итоговой суммы
- Корзина сохраняется в БД

### 5.5 Оформление заказа

- Форма оформления заказа:
    - ФИО получателя
    - адрес доставки
    - номер телефона
    - комментарий к заказу
- Способ оплаты: наличными при получении
- Подтверждение заказа с итоговым чеком

### 5.6 Заказы

- Страница «Мои заказы» для покупателя:
    - список заказов с датой, суммой, статусом
    - детальный просмотр заказа
- Статусы заказа: `Новый` → `Подтверждён` → `В обработке` → `Отправлен` → `Доставлен` → `Отменён`

### 5.7 Профиль покупателя

- Просмотр и редактирование личных данных (имя, адрес доставки)
- История заказов

### 5.8 Панель администратора

#### Товары
- Список всех товаров с поиском
- Создание товара: название, описание, цена, категория, бренд, характеристики, галерея фото, количество на складе
- Редактирование товара
- Удаление товара (мягкое удаление)

#### Категории и бренды
- Создание, редактирование, удаление категорий
- Создание, редактирование, удаление брендов

#### Заказы
- Список всех заказов с фильтрацией по статусу и дате
- Детальный просмотр заказа
- Смена статуса заказа вручную

#### Пользователи
- Список пользователей
- Просмотр профиля пользователя

## 6. Нефункциональные требования

### 6.1 Производительность
- Кэширование часто запрашиваемых данных через Upstash Redis (каталог, категории)
- Server-side rendering (SSR) и статическая генерация (SSG) через Next.js там, где применимо
- Оптимизация изображений через `next/image`

### 6.2 Безопасность
- Защита API-роутов через middleware (проверка JWT и роли)
- Валидация входных данных на сервере (zod)
- HTTPS в продакшне (автоматически через Vercel)

### 6.3 Адаптивность
- Полная адаптивность под мобильные устройства (320px и выше)
- Мобильное меню / бургер-меню
- Адаптивная сетка каталога (1 / 2 / 3 / 4 колонки в зависимости от ширины экрана)

### 6.4 UX
- Состояния загрузки (скелетоны / спиннеры)
- Уведомления об успехе / ошибке (toast-уведомления)
- Пустые состояния (нет товаров, пустая корзина и т.д.)
- 404 страница

## 8. Схема базы данных (основные сущности)

- **User** — пользователь (id, phone, name, role, createdAt)
- **Product** — товар (id, name, description, price, stock, categoryId, brandId, images[], specs, deletedAt)
- **Category** — категория (id, name, slug)
- **Brand** — бренд (id, name, logo)
- **Cart** — корзина (id, userId)
- **CartItem** — позиция в корзине (id, cartId, productId, quantity)
- **Order** — заказ (id, userId, status, total, address, phone, comment, createdAt)
- **OrderItem** — позиция в заказе (id, orderId, productId, quantity, price)

## 10. Что не входит в scope (ограничения)

- Реальная интеграция с SMS-провайдером (mock для разработки, архитектура готова для подключения Messaggio/BudgetSMS через HTTP API)
- Онлайн-оплата через «Клевер» (требует договора с банком-участником системы + интеграция через внутренний эквайринг)
- Многоязычность
- Система отзывов и рейтингов товаров
- Программа лояльности / промокоды
- Статистика и графики в админке
- Блокировка пользователей
