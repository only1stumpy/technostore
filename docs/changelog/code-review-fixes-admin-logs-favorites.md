# Исправления после code review: журнал действий и избранное

## Функциональные изменения

- Все страницы административной панели теперь проходят серверную проверку роли администратора.
- Журнал действий сохраняется при наличии записей и не удаляется каскадно вместе с пользователем-администратором.
- Создание и редактирование товара проверяет, что выбранные категория и бренд активны.
- Категории защищены от циклов в дереве и удаления при наличии активных дочерних категорий.
- API удаления из избранного валидирует параметр товара.
- Кнопка избранного корректнее синхронизирует состояние при обновлении серверных данных.

## Технические изменения

- `app/admin/layout.tsx` вызывает `requireAdmin()` на сервере для всех `/admin` страниц.
- FK `AdminActionLog.adminId` изменен с `ON DELETE CASCADE` на `ON DELETE RESTRICT`.
- Добавлена migration `20260523001000_restrict_admin_action_log_user_delete` для уже примененной базы.
- В admin product routes добавлены проверки активных category/brand перед create/update.
- В admin category route добавлена проверка parent chain перед update и проверка активных children перед delete.
- В favorites routes добавлена Zod-валидация параметров и обработка некорректного JSON как `400`.
- `FavoriteButton` обновлен без синхронного `setState` внутри effect, чтобы проходить React Compiler lint.
- Вывод metadata в журнале действий ограничен по длине, а ошибка записи audit log содержит контекст действия.
