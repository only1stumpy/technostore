# Действия с заказами

## Функциональные изменения

- Покупатель может отменить заказ со статусом `NEW` со страницы деталей заказа.
- После отмены статус заказа меняется на `CANCELLED`, а страница показывает обновленное состояние без ручной перезагрузки.
- При отмене заказа товары возвращаются на склад.
- Если заказ был создан с промокодом, счетчик использований промокода уменьшается обратно.
- Покупатель может повторить любой свой заказ.
- Повтор заказа очищает текущую корзину, добавляет товары из выбранного заказа и перенаправляет пользователя в корзину.
- Повтор заказа проверяет, что товары не удалены и доступного остатка хватает для выбранного количества.
- В деталях заказа отображаются сумма товаров, скидка и примененный промокод.

## Технические изменения

- `OrderRepository` получил методы `cancelByUser()` и `repeatForUser()`.
- `OrderService` пробрасывает действия как `cancelOrder()` и `repeatOrder()`.
- Добавлены пользовательские endpoints:
  - `POST /api/orders/[id]/cancel`;
  - `POST /api/orders/[id]/repeat`.
- Отмена заказа выполняется транзакционно через `updateMany` с условием по текущему статусу `NEW`, чтобы защититься от гонок.
- Возврат складских остатков и корректировка `PromoCode.usedCount` выполняются в той же транзакции, что и смена статуса.
- Повтор заказа использует транзакцию: создает корзину при необходимости, очищает текущие позиции и добавляет позиции из заказа после проверки доступности товаров.
- Страница `/orders/[id]` обновляет Zustand cart-store после повтора заказа.

## Журнал действий

Админские логи уже покрывают основные административные операции проекта: CRUD товаров, категорий и брендов, изменение статуса заказа, модерацию отзывов и управление промокодами. Пользовательские действия отмены и повтора заказа не пишутся в `AdminActionLog`, потому что журнал предназначен для аудита действий администратора.
