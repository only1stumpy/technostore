# Промокоды

## Функциональные изменения

- Покупатель может применить промокод на странице оформления заказа.
- Checkout показывает сумму товаров, размер скидки и итоговую сумму к оплате.
- Промокод проверяется по текущей серверной корзине пользователя, а не по сумме с клиента.
- При создании заказа промокод повторно проверяется и сохраняется вместе с заказом.
- В заказе сохраняются `subtotal`, `discountAmount`, итоговый `total` и применённый код.
- Счётчик использований промокода увеличивается после успешного оформления заказа.
- Администратор может создавать, редактировать и отключать промокоды в админ-панели.
- Для промокодов поддерживаются процентные и фиксированные скидки, минимальная сумма заказа, лимит использований, дата начала и дата окончания действия.

## Технические изменения

- Добавлен `PromoCodeRepository` для расчёта скидок, проверки ограничений и CRUD-операций администратора.
- Добавлен `PromoCodeService` как сервисный слой для пользовательского checkout и админки.
- Добавлен endpoint `POST /api/promo-codes/apply` для предварительного применения промокода к корзине авторизованного пользователя.
- Обновлён `POST /api/orders`: схема принимает optional `promoCode`, а транзакция заказа валидирует код, создаёт `OrderPromoCode` и увеличивает `PromoCode.usedCount`.
- Добавлены admin endpoints:
  - `GET /api/admin/promo-codes`;
  - `POST /api/admin/promo-codes`;
  - `PATCH /api/admin/promo-codes/[id]`;
  - `DELETE /api/admin/promo-codes/[id]`.
- `DELETE` выполняет мягкое отключение через `isActive = false`, чтобы не ломать связь промокодов с заказами.
- Админские действия логируются как `promo-code.create`, `promo-code.update` и `promo-code.deactivate`.
- Checkout хранит состояние промокода локально в форме и не добавляет новый global store.
