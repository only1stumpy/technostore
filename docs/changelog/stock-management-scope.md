# Scope: управление остатками

## Решение по функциональному объему

Отдельный административный раздел для оперативного управления остатками не входит в текущий scope проекта.

## Что остается в проекте

- Остаток товара хранится в поле `Product.stock`.
- Администратор задает и изменяет остаток в форме создания и редактирования товара.
- При оформлении заказа остатки уменьшаются транзакционно.
- Создание заказа защищено проверкой `stock >= quantity`, чтобы не продавать больше доступного количества.
- Товары с нулевым или недостаточным остатком обрабатываются в пользовательских сценариях корзины и заказа.

## Что не реализуется отдельным блоком

- отдельная страница `/admin/stock`;
- массовое обновление остатков;
- фильтры только для складского контроля вроде “мало на складе” или “нет в наличии”;
- отдельные складские операции поступления и списания.

## Причина

Для дипломного e-commerce scope достаточно управления остатком через карточку товара и транзакционной проверки при заказе. Отдельная складская админка расширяет проект в сторону inventory management и не является обязательной для текущего функционального объема интернет-магазина.
