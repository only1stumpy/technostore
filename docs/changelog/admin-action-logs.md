# Журнал действий администратора

## Функциональные изменения

- В административной панели добавлен раздел `/admin/action-logs` для просмотра последних действий администраторов.
- В журнале отображаются дата действия, администратор, тип действия, сущность, идентификатор сущности и технические данные операции.
- Изменение статуса заказа теперь фиксируется в журнале с предыдущим и новым статусом.

## Технические изменения

- В Prisma schema добавлена модель `AdminActionLog` со связью с администратором и индексами по администратору, действию, типу сущности и дате создания.
- Добавлен helper `logAdminAction`, который записывает действие администратора и не ломает основной сценарий при ошибке логирования.
- Добавлен API route `GET /api/admin/action-logs` с проверкой роли администратора, пагинацией и фильтрами по администратору, действию, сущности и датам.
- Добавлена страница `/admin/action-logs` и пункт навигации “Журнал” в layout админ-панели.
- API изменения статуса заказа передает текущего администратора в логирование и сохраняет metadata перехода статуса.
