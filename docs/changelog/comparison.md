# Сравнение товаров

## Функциональные изменения

- Добавлен пользовательский раздел `/compare` для сравнения товаров.
- В карточках товаров появилась кнопка добавления в сравнение.
- В шапке сайта добавлена ссылка на сравнение: иконка на desktop и пункт меню на mobile.
- Авторизованный пользователь может добавлять товары в список сравнения и удалять их повторным нажатием.
- Список сравнения ограничен четырьмя товарами.
- Сравнивать можно только товары из одной категории, чтобы характеристики оставались сопоставимыми.
- Таблица сравнения показывает цену, бренд, категорию, наличие и характеристики из `Product.specs`.
- Отличающиеся значения в таблице визуально выделяются.
- Неавторизованный пользователь при попытке добавить товар в сравнение перенаправляется на страницу входа с параметром возврата.

## Технические изменения

- Добавлена модель `ProductComparison` в Prisma schema и миграции.
- Добавлены DTO `ComparisonItem` и `ComparisonResponse` в `types/api.ts`.
- Добавлен `ComparisonRepository` для работы со списком сравнения в базе данных.
- Добавлен `ComparisonService` и интерфейсы repository/service слоя.
- Добавлены API routes:
  - `GET /api/compare`;
  - `DELETE /api/compare`;
  - `POST /api/compare/items`;
  - `DELETE /api/compare/items/[productId]`.
- API routes используют серверную авторизацию и Zod-валидацию входных данных.
- Повторное добавление товара в сравнение работает идемпотентно.
- Удаление товара из сравнения использует `deleteMany`, поэтому отсутствие записи не ломает пользовательский сценарий.
- Репозиторий исключает soft-deleted товары, категории и бренды из списка сравнения.
- Добавлен клиентский компонент `CompareButton` с optimistic UI и откатом состояния при ошибке.
- Добавлен компонент `ComparisonTable`, который динамически строит строки сравнения из JSON-характеристик товаров.
- `ProductCard` обновлен так, чтобы интерактивные кнопки сравнения и избранного не находились внутри ссылки на товар.

## Улучшение UX и скорости

- Убрано обновление страницы через `router.refresh()` после каждого клика по кнопке сравнения.
- Добавлен Zustand-store для мгновенного optimistic UI и синхронизации с API сравнения.
- Добавлена нижняя sticky-панель сравнения с количеством выбранных товаров, мини-карточками, удалением, очисткой и переходом к таблице.
- Добавлена фоновая инициализация списка сравнения при загрузке приложения.
- Страница `/compare` стала интерактивной: можно удалять товары, очищать список и включать режим “Только отличия” без полной перезагрузки.
- Таблица сравнения получила sticky-колонку характеристик, кнопки удаления товаров и более заметные карточки товаров в заголовке.
- Ошибки лимита, несовпадения категории и авторизации теперь сохраняются в store и показываются пользователю в панели/странице сравнения.
