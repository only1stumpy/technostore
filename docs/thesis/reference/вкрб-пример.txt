22	В данной дипломной работе разработана рекомендательная система для выбора города для переезда на основе предпочтений пользователя. Целью работы является автоматизация процесса подбора локации с учётом индивидуальных приоритетов.

24	ABSTRACT
25	This graduation thesis presents a recommendation system for selecting a city for relocation based on user preferences. The aim of the work is to automate the process of choosing a location considering individual priorities.
26	Three multi-criteria decision-making methods (WSM, TOPSIS, CBR), city filtering, comparison, search history and favorites are implemented. The developed application enables informed relocation decisions and can be used by individuals and HR specialists.

35	1 ИССЛЕДОВАНИЕ И АНАЛИЗ ПРЕДМЕТНОЙ ОБЛАСТИ
36	1.1 Описание поставленной задачи, ее обоснование
37	Процесс выбора города для переезда представляет собой сложную многогранную задачу, с которой сталкиваются различные категории лиц: специалисты, рассматривающие варианты трудовой миграции; студенты, выбирающие место для получения образования; семьи с маленькими детьми, ищущие безопасную и стабильную среду; пенсионеры, стремящиеся к комфортным условиям для жизни, а также digital-кочевники (цифровые кочевники), которые могут работать удаленно из любой точки мира.
38	Основная сложность данной задачи заключается в том, что выбор оптимального города требует одновременного учета множества разнородных факторов. К числу таких факторов относятся экономические показатели (уровень доходов, стоимость жизни, доступность жилья), социальные индикаторы (безопасность, качество здравоохранения, образование), а также субъективные аспекты (климатические условия, языковой барьер, культурная среда) и многое другое.
39	Ключевая проблема, выявленная в ходе анализа предметной области, заключается в том, что существующие инструменты для подбора города переезда, как правило, предлагают универсальные рейтинги, основанные на фиксированных весах критериев. Такой подход не позволяет учесть индивидуальные предпочтения пользователя, которые могут существенно отличаться: для одного пользователя наиболее важным фактором может быть безопасность, для второго низкая стоимость жизни, для третьего качество здравоохранения.
40	Исходя из этого, задача, решаемая в рамках данной дипломной работы, формулируется следующим образом: разработка информационной системы, которая на основе предпочтений пользователя, выраженных в виде весовых коэффициентов или ранжирования критериев, формирует персонализированные рекомендации по выбору локации для переезда, а также предоставляет возможность сравнения городов по ключевым показателям качества жизни.
41	1.2 Обоснование актуальности исследуемой задачи
42	Актуальность разработки рекомендательной системы для планирования переезда обусловлена несколькими взаимосвязанными факторами.
43	Во-первых, глобализация и цифровая трансформация экономики вызвали значительный рост географической мобильности населения. Возможность удаленной работы, развитие международных транспортных коммуникаций и унификация требований к специалистам в различных странах создали условия для того, чтобы подбор места жительства стал осознанным стратегическим решением, а не вынужденной необходимостью.
44	Во-вторых, информационная перегрузка, характерная для современного этапа развития общества, формирует дополнительные барьеры для принятия обоснованных решений. Потенциальный мигрант сталкивается с необходимостью анализа большого объёма разнообразной информации из различных источников: международных рейтингов (Numbeo, Transparency International, World Happiness Report, EF English Proficiency Index, QS World University Rankings, Heritage Foundation, Social Progress Imperative, статистических данных Международной организации труда (ILO)), статистических отчетов национальных агентств, отзывов других пользователей и экспертных заключений. Синтез этой информации в единую картину является трудоемкой задачей, решение которой требует значительных временных затрат и специальных знаний.
45	В-третьих, существующие инструменты для оценки городов (Numbeo, Nomadlist, The Earth Awaits, Expatistan, Livingcost) предоставляют пользователю обобщенные данные о стоимости, безопасности или качестве медицинского обслуживания. Однако эти сервисы, как правило, не принимают во внимание индивидуальные приоритеты пользователя и не предоставляют механизмов персонализации рекомендаций.
46	В-четвертых, задача выбора места жительства по множеству критериев формально относится к классу задач многокритериального принятия решений (Multi-Criteria Decision Making, MCDM). Исследования в области MCDM показывают, что использование специализированных методов, таких, как метод взвешенной суммы (Weighted Sum Model, WSM), метод анализа иерархий (Analytic Hierarchy Process, AHP), метод TOPSIS (Technique for Order Preference by Similarity to Ideal Solution) и другие позволяют существенно повысить обоснованность и объективность принимаемых решений. Тем не менее практическая реализация этих методов в виде доступных программных инструментов для конечных пользователей остается ограниченной.
47	Таким образом, разработка рекомендательной системы, сочетающей в себе достоверную базу данных о городах с методами многокритериального анализа и возможностью персонализации, представляет собой актуальную научно-практическую задачу.
48	1.3 Современное состояние исследуемой задачи
49	Для системного анализа современного состояния исследуемой задачи целесообразно рассмотреть три взаимосвязанных аспекта. Во-первых, необходимо проанализировать существующие международные рейтинги городов, которые являются основным источником информации о качестве жизни и формируют общепринятые представления о «лучших» городах для проживания. Во-вторых, следует рассмотреть научные методы многокритериального принятия решений, которые лежат в основе современного подхода к решению задач выбора при наличии множества противоречивых критериев. В-третьих, необходимо провести обзор существующих информационных систем и онлайн-сервисов, предоставляющих данные о городах и/или рекомендации по выбору места жительства. Такое структурирование позволяет выявить пробелы в существующих решениях и обосновать необходимость разработки новой системы.
50	Анализ международных рейтингов городов и индексов качества жизни:
51	В разрабатываемой рекомендательной системе используется комбинация данных из семи авторитетных источников, каждый из которых оценивает определённый аспект качества жизни. Такой мульти-источниковый подход позволяет составить комплексную и объективную картину привлекательности городов для переезда. Ниже представлен детальный анализ каждого из используемых рейтингов и индексов:
52	А) Numbeo (агрегатор данных о качестве жизни):
53	Numbeo представляет собой крупнейшую краудсорсинговую базу данных о качестве жизни в городах мира. Особенностью Numbeo является то, что данные формируются на основе вкладов самих пользователей жителей и посетителей городов. Этот подход обеспечивает высокую актуальность информации (данные обновляются постоянно) и позволяет получить «народную» оценку, отражающую реальный опыт людей. Numbeo предоставляет следующие индексы, используемые в разрабатываемой системе:
54	Индекс покупательной способности (Purchasing Power Index) отражает относительную способность местных жителей приобретать товары и услуги с учётом среднего уровня доходов;
55	Индекс безопасности (Safety Index) оценивает уровень преступности и ощущение защищённости жителей;
56	Индекс здравоохранения (Health Care Index) характеризует качество, доступность и стоимость медицинских услуг;
57	Индекс климата (Climate Index) оценивает комфортность климатических условий (температура, влажность, количество солнечных дней);
58	Индекс стоимости жизни (Cost of Living Index) показывает относительную стоимость потребительской корзины;
59	Соотношение цены недвижимости к доходу (Property Price to Income Ratio) характеризует доступность жилья для местных жителей;
60	Индекс загрязнения (Pollution Index) оценивает уровень загрязнения воздуха, воды и шумовое загрязнение.
61	Все индексы Numbeo нормированы в диапазоне от 0 до 100 (за исключением соотношения цены недвижимости к доходу, которое может превышать 100), что упрощает их дальнейшую обработку. Numbeo выбран в качестве основного источника данных благодаря широте охвата (более 5000 городов мира), регулярному обновлению и доступности информации [5].
62	Б) Transparency International (Индекс восприятия коррупции):
63	Transparency International (Международная организация по борьбе с коррупцией) ежегодно публикует Индекс восприятия коррупции (Corruption Perception Index, CPI). Данный индекс оценивает уровень коррупции в государственном секторе на основе экспертных оценок и опросов бизнес-аналитиков. Индекс рассчитывается по шкале от 0 до 100, где 0 соответствует максимальному уровню коррупции, а 100 минимальному. В разрабатываемой системе данный индекс используется как показатель «чистоты» государственных институтов и прозрачности ведения бизнеса в стране, что является важным фактором для планирования долгосрочного переезда [6].
64	В) World Happiness Report (Всемирный рейтинг счастья населения):
65	Всемирный рейтинг счастья населения (World Happiness Report) публикуется по инициативе Организации Объединённых Наций и ранжирует страны по уровню субъективного благополучия населения. Доклад основан на данных глобальных опросов Gallup World Poll и учитывает шесть ключевых факторов: ВВП на душу населения, социальную поддержку, ожидаемую продолжительность здоровой жизни, свободу принимать жизненные решения, щедрость (уровень благотворительности) и восприятие коррупции. В разрабатываемой системе используется позиция страны в глобальном рейтинге счастья, где более низкое числовое значение (место) соответствует более счастливому обществу [7].
66	Г) EF EPI (Индекс владения английским языком):
67	EF English Proficiency Index (EF EPI) ежегодный рейтинг, публикуемый образовательной компанией EF Education First, который оценивает уровень владения английским языком в различных странах мира. Рейтинг основан на результатах стандартизированного тестирования миллионов взрослых людей. Страны ранжируются по шкале от «Очень высокий» до «Очень низкий» с присвоением числового значения − места в рейтинге. Для человека, рассматривающего переезд в другую страну, уровень владения местным населением английским языком является критически важным фактором, особенно если он не владеет местным языком [8].
68	Д) QS World University Rankings (рейтинг университетов):
69	QS World University Rankings  один из наиболее авторитетных международных рейтингов высших учебных заведений, публикуемый компанией Quacquarelli Symonds. Рейтинг оценивает университеты по таким критериям, как академическая репутация, репутация среди работодателей, соотношение преподавателей к студентам, индекс цитирования научных работ и доля иностранных студентов. В разрабатываемой системе используется позиция лучшего университета страны в глобальном рейтинге как индикатор качества системы образования. Более низкое числовое значение (место) соответствует более качественному образованию, что особенно важно для семей с детьми и студентов [9].
70	Е) Heritage Foundation (Индекс экономической свободы):
71	Индекс экономической свободы (Index of Economic Freedom) ежегодно публикуется аналитическим центром Heritage Foundation в партнёрстве с The Wall Street Journal. Индекс оценивает страны по 12 критериям, сгруппированным в четыре категории: верховенство закона (права собственности, свобода от коррупции), размер государства (налоговая нагрузка, государственные расходы), эффективность регулирования (свобода бизнеса, свобода труда, свобода от чрезмерного регулирования) и открытость рынков (свобода торговли, инвестиционная свобода, финансовая свобода). Страны получают общую оценку от 0 до 100, где более высокий балл соответствует более высокой степени экономической свободы. В разрабатываемой системе используется позиция страны в рейтинге экономической свободы, что позволяет оценить лёгкость ведения бизнеса и предпринимательский климат [10].
72	Ж) Social Progress Imperative (Индекс социальной защиты):
73	Индекс социальной защиты (Social Progress Index, SPI) публикуется некоммерческой организацией Social Progress Imperative. В отличие от большинства экономических показателей, SPI фокусируется исключительно на социальных и экологических результатах, а не на экономических входах (например, ВВП). Индекс оценивает страны по трём основным измерениям: базовые человеческие потребности (питание, вода, жильё, безопасность), основы благополучия (доступ к базовым знаниям, доступ к информации и коммуникациям, здоровье и санитария, качество окружающей среды) и возможности (личные права, свобода выбора, толерантность и инклюзивность, доступ к высшему образованию). В разрабатываемой системе данный индекс используется для комплексной оценки социальной защиты и качества жизни, которые сложно измерить экономическими показателями [11].
74	Ключевое ограничение всех рассмотренных источников с точки зрения задачи персонализации заключается в том, что каждый из них предоставляет «усреднённую» оценку, не учитывающую индивидуальные предпочтения пользователя. Например, Numbeo рассчитывает общий индекс качества жизни с фиксированными весами критериев, а World Happiness Report использует собственную методику взвешивания шести факторов. Именно это ограничение определяет необходимость разработки системы, которая позволяет пользователю самостоятельно задавать веса критериев в соответствии со своими приоритетами.
75	Многокритериальное принятие решений как научная основа:
76	Многокритериальное принятие решений представляет собой область исследования, которая занимается разработкой и применением методов для поддержки принятия решений в ситуациях, когда необходимо оценить альтернативы по нескольким, часто противоречивым, критериям. Формально задача MCDM может быть описана следующим образом: имеется m альтернатив (объектов) A1, A2, …, AM, каждая из которых характеризуется n признаками (критериями) C1, C2, …, CN. Задача заключается в выборе наилучшей альтернативы или ранжировании всех альтернатив на основе оценки их признаков с учетом предпочтений лица, принимающего решение [1].
77	В современной теории MCDM методы классифицируются на следующие основные группы:
78	А) Методы измерения ценности (Value Measurement methods) к этой группе относятся WSM, Ratio System (RS), Weighted Product Model (WPM), Multi-Attributive Border Approximation Area Comparison (MABAC). Данные методы основаны на преобразовании вектора признаков каждой альтернативы в скалярную оценку (рейтинг) с использованием аддитивной или мультипликативной функции агрегирования;
79	Б) Модели целевого или референсного уровня (Goal or Reference Level Models) к этой группе относятся TOPSIS и VIKOR. Эти методы базируется на сравнении альтернатив с идеальным решением (наилучшим по всем критериям) и антиидеальным решением (наихудшим по всем критериям);
80	В) Методы превосходства (Outranking Techniques) к этой группе относятся PROMETHEE и ELECTRE. Данные методы строятся на попарном сравнении альтернатив и построении отношения превосходства.
81	Современным трендом в области MCDM является синтез решений на основе множества моделей (Multi-Method Model, 3M), что предполагает, что решение, основанное на применении нескольких методов, повышает надежность принимаемых решений. В соответствии с этим подходом результаты, полученные различными методами, приводятся к единой шкале и агрегируются, что позволяет получить более информативный рейтинг по сравнению с задействованием отдельного метода [2], [3], [4].
82	Существующие информационные системы для выбора города:
83	В настоящее время на рынке представлен ряд информационных ресурсов, предоставляющих пользователям данные о качестве жизни в различных городах мира. Анализ этих ресурсов позволяет выделить следующие наиболее значимые платформы.
84	Numbeo  крупнейшая база данных о стоимости жизни и качестве жизни. Основные характеристики: данные основаны на пользовательских вкладах; предоставляет индексы по 10+ категориям; доступен API для программного доступа. Недостаток: отсутствие персонализации рекомендаций.
85	Nomadlist  платформа, ориентированная на цифровых кочевников. Специализируется на городах, подходящих для удаленной работы; учитывает скорость интернета, визовую политику, стоимость коворкингов. Недостаток: данные часто устаревают, методы оценки непрозрачны.
86	The Earth Awaits  сервис с фильтрацией городов по бюджету. Использует данные Numbeo; позволяет задать бюджет, размер семьи, тип жилья; выдает список городов, соответствующих критериям. Недостаток: отсутствие весовой настройки критериев, только фильтрация.
87	Expatistan  альтернативный источник данных о стоимости жизни. Данные основаны на пользовательских вкладах; предлагает сравнение стоимости жизни между городами. Недостаток: ограниченный набор критериев, отсутствие рекомендательной функции.
88	Сравнительный анализ показывает, что ни одна из существующих систем не предоставляет персонализацию рекомендаций на основе весовых коэффициентов критериев, возможность выбора метода расчета (WSM, TOPSIS, CBR), сравнение городов по всем критериям с определением лучшего по каждому критерию, а также сохранение истории поисков и избранного.
89	1.4 Обзор методов решения подобных задач
90	Для решения задачи выбора оптимального города по множеству критериев в рамках данной работы были рассмотрены три метода многокритериального анализа, различающиеся по принципу работы, сложности реализации и способу представления пользовательских предпочтений: WSM (Weighted Sum Model), TOPSIS (Technique for Order Preference by Similarity to Ideal Solution) и CBR (Case-Based Reasoning).
91	Метод взвешенной суммы (WSM):
92	Метод взвешенной суммы является одним из наиболее простых и интуитивно понятных методов многокритериального анализа. Основная идея метода заключается в вычислении интегральной оценки для каждой альтернативы как взвешенной суммы нормализованных значений критериев:
93	Si=j=1nwj∙rij ,
94	где Si – интегральная оценка i-й альтернативы, wj – вес j-го критерия (wj=1), rij – нормализованное значение j-го критерия для i-й альтернативы.
95	Нормализация критериев:
96	Поскольку критерии имеют различные единицы измерения и шкалы, а также разнонаправленный характер, в разработанной системе применяется процедура нормализации, приводящая все значения к единому диапазону [0,1] [0,1]. Важной особенностью является использование фиксированных референсных значений (максимальных наблюдаемых показателей) [1] вместо динамических максимумов по выборке, что обеспечивает стабильность результатов при изменении набора городов [1].
97	Для каждого критерия используется своя функция нормализации, учитывающая его физический смысл:
98	Благоприятные критерии (beneficial) − большее значение соответствует лучшему качеству. Нормализация выполняется по формуле:
99	rj=min1, xjrefjmax,
100	где refjmax – максимальное наблюдаемое значение критерия [1].
101	Неблагоприятные критерии (non-beneficial) − меньшее значение соответствует лучшему качеству. Нормализация выполняется по формуле:
102	rj=max(0,min1, 1-xjrefjmax)
103	Рейтинговые критерии, где значение представляет собой место в глобальном рейтинге (1 − лучший результат), нормализуются по формуле:
104	rj=max⁡(0,min1, 1- rankj-1rankmax-1)
105	Преимущества метода WSM в контексте разрабатываемой системы:
106	Простота реализации и интерпретации − пользователю интуитивно понятен принцип работы.
107	Прозрачность влияния весов − изменение веса критерия приводит к предсказуемому изменению итоговой оценки.
108	Высокая скорость вычислений − метод имеет сложность O (m · n), где m − количество городов, n − количество критериев.
109	Стабильность результатов − использование фиксированных референсов обеспечивает независимость ранжирования от состава выборки.
110	Недостатки метода WSM:
111	Полная компенсация − метод предполагает, что низкие значения по одним критериям могут быть полностью скомпенсированы высокими значениями по другим, что может приводить к выбору альтернативы с экстремально низким значением по критически важному критерию.
112	Зависимость от нормализации − результат чувствителен к способу нормализации данных.

Все эти источники используют фиксированные методики расчёта, что не позволяет учесть индивидуальные предпочтения пользователя, что является ключевым ограничением существующих рейтингов.
195	Научной основой разрабатываемой системы являются методы многокритериального принятия решений (MCDM). Анализ литературы показал, что наиболее применимыми для задачи выбора города являются:
196	Метод взвешенной суммы (WSM) − для точного задания весов критериев с последующей линейной нормализацией;
197	Метод TOPSIS − для задания предпочтений через ранжирование критериев с расчётом весов по экспоненциальному закону;
198	Метод рассуждений на основе прецедентов (CBR) − для оценки улучшения качества жизни относительно текущего места проживания.
199	Анализ существующих информационных систем (Numbeo, Nomadlist, The Earth Awaits, Expatistan, Livingcost) показал, что ни одна из них не предоставляет персонализации рекомендаций на основе индивидуальных предпочтений с возможностью настройки весов критериев, выбора метода расчёта и сохранения истории поисков. Выявленное отсутствие является основным обоснованием целесообразности разработки новой системы.

229	2.2 Описание применяемых алгоритмов
230	Алгоритм WSM (Weighted Sum Model):
231	Метод взвешенной суммы реализован в функции calculate_wsm_score файла recommendation_engine.py. Алгоритм включает следующие этапы:
232	Шаг 1. Нормализация критериев:
233	Для каждого критерия выполняется нормализация к единой шкале [0, 1] с использованием фиксированных референсных значений:
234	Для beneficial (чем больше, тем лучше):
235	norm=min(1, valuemax_ref)
236	Для non-beneficial (чем меньше, тем лучше):
237	norm=min(1, 1-valuemax_ref)
238	Для рейтинговых (место в рейтинге, 1 – лучший):
239	norm=1- value-1max⁡_rank-1
240	где max_ref – максимальное наблюдаемое значение для данного критерия (например, для безопасности – 93.24, для стоимости жизни – 118.5), max_rank – максимальное место в рейтинге.
241	Шаг 2. Взвешенное суммирование:
242	Интегральная оценка города вычисляется как взвешенная сумма нормализованных значений:
243	score= (normj ∙ weightj) ,
244	где weightj – нормализованный вес критерия (сумма всех весов = 1).
245	Шаг 3. Сортировка результатов:
246	Города сортируются по убыванию полученной оценки. Результат представляется пользователю в процентах (score × 100%).
247	Алгоритм TOPSIS (Technique for Order Preference by Similarity to Ideal Solution):
248	Метод TOPSIS реализован в функции topsis_calculate и основан на сравнении каждой альтернативы с идеальным и антиидеальным решениями.
249	Шаг 1. Построение матрицы решений:
250	Для каждого города формируется вектор из 14 нормализованных значений критериев.
251	Шаг 2. Векторная нормализация:
252	Каждый элемент матрицы делится на евклидову норму соответствующего столбца:
253	rij=xijxij2
254	Шаг 3. Взвешивание нормализованной матрицы:
255	Каждый элемент умножается на вес соответствующего критерия:
256	vij=rij∙wij
257	Шаг 4. Определение идеального и антиидеального решений:
258	Для благоприятных критериев (beneficial): идеальное значение = максимум в столбце, антиидеальное = минимум;
259	Для неблагоприятных критериев (non-beneficial): идеальное значение = минимум, антиидеальное = максимум.
260	Шаг 5. Расчёт расстояний
261	Евклидово расстояние от каждой альтернативы до идеального S+ и антиидеального S- решений:
262	Si+=(vij-idealj)2
263	Si-=(vij-antiidealj)2
264	Шаг 6. Вычисление относительной близости:
265	Ci=Si-(Si++Si-)
266	где Ci – итоговая оценка города в диапазоне [0, 1].
267	Алгоритм CBR (Case-Based Reasoning):
268	Метод рассуждений на основе прецедентов реализован в функциях calculate_delta_score и cbr_recommendation и отвечает на вопрос: «Насколько переезд в город Y улучшит качество жизни по сравнению с текущим городом X?».
269	Шаг 1. Поиск текущего города:
270	Текущий город пользователя ищется в базе данных по названию и стране с нормализацией строк (приведение к нижнему регистру, удаление диакритических знаков).
271	Шаг 2. Расчёт направленного улучшения по каждому критерию:
272	Для каждого критерия вычисляется дельта-оценка:
273	Для благоприятных критериев (beneficial):
274	improvement=candidatenorm-currentnorm
275	Для неблагоприятных критериев (non-beneficial):
276	improvement=currentnorm-candidatenorm
277	Шаг 3. Взвешенное суммирование:
278	deltascore=improvementj∙weightj
279	где weightj – нормализованный приоритет улучшения, заданный пользователем.
280	Шаг 4. Формирование результата:
281	Результат представляется в виде процента улучшения:
282	improvementpercent=deltascore∙100
283	Положительное значение означает, что город-кандидат лучше текущего, отрицательное – хуже.
284	Алгоритм фильтрации городов:
285	Фильтрация городов реализована в функции apply_country_filter и поддерживает шесть типов фильтров:
286	include – оставить только указанные страны;
287	exclude – исключить указанные страны;
288	region – оставить только страны из указанных регионов;
289	region_exclude – исключить страны из указанных регионов;
290	city_exclude – исключить конкретные города;
291	city_include – оставить только указанные города.

395	Расчёт для каждого города:
396	Евклидова расстояния до идеального решения;
397	Евклидова расстояния до антиидеального решения.
398	Вычисление относительной близости как: расстояние до антиидеала / (расстояние до идеала + расстояние до антиидеала).
399	Сортировка городов по убыванию полученных оценок.
400	Результат интерпретируется как степень близости города к идеальному решению (от 0% до 100%).
401	Реализация метода CBR:
402	Метод рассуждений на основе прецедентов оценивает, насколько переезд в город-кандидат улучшит качество жизни по сравнению с текущим городом пользователя. Алгоритм состоит в следующем:
403	Получение текущего города пользователя (выбор из выпадающего списка).
404	Получение приоритетов улучшения (целые числа 0-100 для каждого критерия).
405	Поиск данных о текущем городе в базе данных
406	Для каждого города-кандидата (кроме текущего):
407	Для каждого критерия, по которому есть данные в обоих городах:
408	Расчёт нормализованных значений для текущего и кандидата;
409	Вычисление направленного улучшения:
410	Для благоприятных критериев: (норма кандидата – норма текущего);
411	Для неблагоприятных критериев: (норма текущего – норма кандидата);
412	Умножение улучшения на вес приоритета (после нормализации весов);
413	Суммирование взвешенных улучшений по всем критериям.
414	Сортировка городов по убыванию суммарной дельта-оценки.
415	Представление результата как процент улучшения (дельта-оценка × 100%).
416	Положительное значение означает, что город-кандидат лучше текущего, отрицательное – хуже.
417	Реализация фильтрации городов:
418	Фильтрация применяется к списку городов перед выполнением расчётов. Поддерживаются следующие типы фильтров:
419	Без фильтра – отображаются все города;
420	Включение стран – отображаются только города из выбранных стран;
421	Исключение стран – скрываются города из выбранных стран;
422	Включение регионов – отображаются города из стран, входящих в выбранные регионы (Европа, Азия, Северная Америка, Южная Америка, Африка, Австралия и Океания);
423	Исключение регионов – скрываются города из стран выбранных регионов;
424	Исключение городов – скрываются конкретные города (пары «город, страна»).
425	Хранение данных:
426	База данных PostgreSQL содержит три основные таблицы:
427	cities_quality_data – хранит для каждого города 14 критериев качества жизни, источник данных и дату последнего обновления.
428	search_history – хранит историю поисков пользователей (метод расчёта, веса критериев, настройки, результаты, дату создания).
429	quality_cache_stats – хранит статистику использования кэша (количество запросов, попаданий и промахов).
430	Избранное хранится в JSON-файле favorites_storage.json на стороне Python-ядра. Каждый пользователь идентифицируется по уникальному иденти-фикатору, генерируемому на клиенте и сохраняемому в localStorage браузера.
431	История поисков идентифицируется по сессии – на основе IP-адреса и User-Agent клиента формируется хэш, который используется для привязки записей к пользователю.

558	Выявленные проблемы и их устранение:
559	Проблема 1. При отсутствии данных по какому-либо критерию (значение 0.00 в Numbeo) метод WSM мог выдавать заниженные оценки, так как нормализация возвращала 0.
560	Решение: В функцию has_valid_data добавлена проверка: если значение равно 0.00 или пустой строке, оно считается отсутствующим. Такие критерии исключаются из расчёта.
561	Проблема 2. При фильтрации по регионам использовались английские названия (europe, asia и т.д.), а в базе данных – русские названия стран.
562	Решение: Создан словарь REGIONS, где для каждого региона перечислены все страны этого региона на русском языке. Фильтрация выполняется по этим спискам.
563	Проблема 3. При большом количестве городов (более 500) TOPSIS работал медленно из-за полного перебора.
564	Решение: Оптимизирована работа с массивами – вместо обычных списков Python используются векторизованные операции NumPy, что ускорило вычисления примерно в 3 раза.
565	Проблема 4. История поисков идентифицировалась только по IP-адресу, что приводило к смешиванию данных разных пользователей из одной сети (например, офисная сеть).
566	Решение: Добавлен дополнительный параметр – User-Agent. Для идентификации используется хэш от IP-адреса и User-Agent вместе.
567	Оценка надежности:
568	Расчёт коэффициента готовности (Availability):
569	Система состоит из трёх компонентов: клиент (React), сервер (Node.js), вычислительное ядро (Python). Предполагая, что каждый компонент имеет надёжность 0,99 (99% времени работает без сбоев), коэффициент готовности системы составляет:
570	А=0.99 ∙0.99 ∙0.99=0.9703 (≈97%)
571	Время наработки на отказ (MTBF):
572	В процессе тестирования (1000 итераций запросов) было зафиксировано 2 сбоя (оба связаны с временной недоступностью Python-ядра). Таким образом, MTBF составляет:
573	MTBF= 10002=500 (запросов на отказ)

596	При работе с разработанной рекомендательной системой на рабочем месте программиста и конечного пользователя имеют место следующие вредные и опасные факторы в соответствии с ГОСТ 12.0.003-2015 [19]:
597	Повышенный уровень электромагнитного излучения – воздействие от монитора и системного блока.
598	Недостаточная освещённость рабочей зоны – может привести к снижению остроты зрения и повышенной утомляемости.
599	Повышенная нагрузка на зрение – длительная работа с монитором (статическое напряжение аккомодации глаза).
600	Статическая нагрузка на опорно-двигательный аппарат – длительное нахождение в позе сидя.
601	Психоэмоциональное напряжение – при работе с большими объёмами данных и принятии ответственных решений.

611	Перерыв на 10–15 минут после каждого часа работы;

633	В результате выполнения выпускной квалификационной работы была разработана рекомендательная система для выбора города для переезда на основе предпочтений пользователя.
634	Основные результаты работы:
635	Проведён анализ предметной области – выявлены основные проблемы существующих систем (отсутствие персонализации, фиксированные веса критериев). Обоснована актуальность разработки.
636	Разработана архитектура системы – трёхуровневая архитектура (клиент на React, API Gateway на Node.js, вычислительное ядро на Python) обеспечивает модульность и масштабируемость.
637	Реализованы три метода многокритериального анализа:
638	WSM – для точного задания весов критериев;
639	TOPSIS – для задания предпочтений через ранжирование;
640	CBR – для оценки улучшения относительно текущего города.
641	Реализованы дополнительные функции:
642	Фильтрация городов по странам, регионам и отдельным городам;
643	Сохранение истории поисков с возможностью переименования и загрузки;
644	Ведение списка избранных городов;
645	Интерактивное сравнение городов с подсветкой лучших значений.
646	Сформирована база данных – более 500 городов с 14 критериями качества жизни из 7 авторитетных источников.
647	Проведено тестирование – система успешно прошла модульное, интеграционное и приёмочное тестирование.
648	Практическая значимость работы заключается в создании инструмента, позволяющего пользователю принимать обоснованные решения о переезде с учётом индивидуальных предпочтений. Система может быть использована частными лицами для планирования переезда, а также может быть интегрирована в корпоративные HR-процессы для поддержки сотрудников при релокации.
